\doxysubsection{Detailed Description}
The core volumetric actor that simulates dynamic smoke expansion using a deterministic voxel-\/based flood-\/fill algorithm.\hypertarget{class_a_i_v_smoke_voxel_volume_autotoc_md4}{}\doxysubsubsection{\texorpdfstring{Overview}{Overview}}\label{class_a_i_v_smoke_voxel_volume_autotoc_md4}
This actor generates a 3D grid of voxels that expand outward from the center, navigating around obstacles defined by the collision settings. The simulation is\texorpdfstring{$\ast$}{*}deterministic, ensuring the same shape and timing across both Server and Clients without replicating individual voxel data.\hypertarget{class_a_i_v_smoke_voxel_volume_autotoc_md5}{}\doxysubsubsection{\texorpdfstring{Simulation Lifecycle}{Simulation Lifecycle}}\label{class_a_i_v_smoke_voxel_volume_autotoc_md5}
The simulation state machine progresses based on the sum of duration and fade settings:\+
\begin{DoxyEnumerate}
\item {\bfseries{Idle}}:\+ Initial state.
\item {\bfseries{Expansion}}:\+ Spawns voxels. Ends after {\ttfamily \doxylink{class_a_i_v_smoke_voxel_volume_a7f6f92c9a2150fd42ed349805395f7f2}{Expansion\+Duration} + \doxylink{class_a_i_v_smoke_voxel_volume_a2f5ed5eb5d3dccaa786795fb1b905943}{Fade\+In\+Duration}}.
\item {\bfseries{Sustain}}:\+ Maintains the shape. Ends after {\ttfamily \doxylink{class_a_i_v_smoke_voxel_volume_a5837d33af8adc2b43930f5c19b1b8d4d}{Sustain\+Duration}}.
\item {\bfseries{Dissipation}}:\+ Removes voxels. Ends after {\ttfamily \doxylink{class_a_i_v_smoke_voxel_volume_a4233c1cdaf810100d2672699b2b96879}{Dissipation\+Duration} + \doxylink{class_a_i_v_smoke_voxel_volume_a7d92753fc9e0a1371b6abee219a07561}{Fade\+Out\+Duration}}.
\item {\bfseries{Finished}}:\+ Simulation complete.
\end{DoxyEnumerate}\hypertarget{class_a_i_v_smoke_voxel_volume_autotoc_md6}{}\doxysubsubsection{\texorpdfstring{Network \& Execution}{Network \& Execution}}\label{class_a_i_v_smoke_voxel_volume_autotoc_md6}
The simulation logic executes deterministically on both the Server and Client.
\begin{DoxyItemize}
\item The Server manages the authoritative state (State, Seed, Start\+Time) and replicates it to Clients.
\item Clients execute the exact same flood-\/fill algorithm locally based on the replicated Seed and Time. 
\end{DoxyItemize}
